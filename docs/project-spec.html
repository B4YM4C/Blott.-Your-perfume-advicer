01System Overview

Description · Actors · External systems

Description

Web quiz สำหรับแนะนำน้ำหอม

User เข้ามาตอบคำถาม 5–10 ข้อ (admin ปรับได้) แล้วระบบแนะนำน้ำหอมที่เหมาะสมที่สุด โดย admin สามารถปรับแต่งคำถาม ช้อย และผลลัพธ์ผ่าน back-office portal — ไม่มีการสมัครสมาชิก ใช้แค่ username นามแฝง + cookie tracking ตามมาตรฐาน PDPA

Actors

เข้ามาเพื่อตอบคำถาม 5–10 ข้อแล้วได้รับคำแนะนำน้ำหอม ไม่ต้องสมัครสมาชิก กรอก username นามแฝงเพื่อให้ระบบเรียก หลังจบ quiz เลือกได้ว่าจะกรอก email เพื่อรับผลทาง email หรือกด Skip

06Quiz Logic

Encoding · concat · matching

Encoding

เมื่อ user ตอบคำถามข้อที่ N ด้วย choice index K → บันทึก {questionOrder: N, choiceCode: indexToLetter(K)}. แยกเก็บเป็น 1 row/question (ตาราง answers) เพื่อให้ admin เพิ่ม/ลด question ได้โดยไม่ต้อง migrate

Concat

เมื่อ submit ครบ → buildPattern() เรียงตาม questionOrder แล้ว concat:

{ q1: A, q2: B, q3: C, q4: D, q5: A } → "1A2B3C4D5A"

Matching

ระบบลอง match กับ result_mappings ตามลำดับ:

  1. Exact — ตรงตัวอักษรทั้งหมด (no *)
  2. Wildcard — match ทีละ position โดย * = any choice
  3. Default — fallback row ที่ pattern = default

07Tracking · PDPA

Cookie consent · audit trail · what we do/don't collect

เก็บข้อมูล

What we collect (when consent = accepted)

  • Username นามแฝง (ไม่ใช่ชื่อจริง — ผู้ใช้พิมพ์เอง)
  • Email (เฉพาะคนที่กรอก ไม่ skip)
  • คำตอบทุกข้อ + pattern ที่เกิดขึ้น
  • Tracking events: page_view, quiz_start, question_answered, quiz_complete
  • Referrer + user-agent (ระดับ browser-family เท่านั้น)
ไม่เก็บ

What we never collect

  • Real name / national ID / phone
  • Precise location (no GPS, no IP geolocation beyond country)
  • Device fingerprint / canvas hashing
  • Third-party cross-site tracking (no Facebook Pixel default)
  • Health, religion, political views — sensitive PDPA categories

Consent flow

  1. First visit → ConsentBanner ปรากฏ (rejected = default until decided)
  2. User clicks Accept / Reject → save localStorage.blott_consent + POST /api/consent
  3. POST is logged ใน consent_log เป็น audit trail พร้อม timestamp
  4. If rejected → /api/tracking calls สั่งระงับ (ทั้งฝั่ง client และ server)
  5. Withdraw ทำได้ตลอดเวลาผ่านหน้า /privacy → log เป็น withdrawn

10How to run

Local (no DB) · Production (SQL)

Local — no database

cd blott-app
cp .env.example .env.local      # APP_MODE=local (default)
npm install
npm run dev                      # http://localhost:3000
                                 # http://localhost:3000/admin

Mock DB จะ seed ข้อมูลจาก data/questions.json และ data/mappings.json โดยอัตโนมัติ — restart Node = reset ทุกอย่าง

Production — with SQL

mysql -u blott -p blott < sql/schema.sql

# .env.local
APP_MODE=production
DB_DRIVER=mysql
DB_HOST=…  DB_USER=…  DB_PASSWORD=…  DB_NAME=blott

npm run build && npm start

Code เดียวกันทำงานทั้ง 2 mode เพราะ lib/db/index.js เป็น proxy ที่ switch ตาม APP_MODE

11Roadmap (post-beta)

Notes for v.02+

  • Real auth for admin (NextAuth + email magic link / SSO)
  • Image uploads via S3 / Cloudinary แทนการกรอก URL
  • A/B testing framework สำหรับเปรียบเทียบคำถามหลายเวอร์ชัน
  • Recommendations engine — เปลี่ยนจาก static mapping เป็น weighted scoring + collaborative filtering เมื่อมีข้อมูลพอ
  • Email provider integration — Resend / SendGrid พร้อม template ที่ผ่าน DKIM/SPF
  • Analytics dashboard ใน admin (top results, drop-off rate, time-per-question)
  • i18n — TH (default) + EN + JP
  • Affiliate links ในหน้า result เพื่อ monetise
