02 · Quiz Logic

How answer becomes perfume

Scoring pipeline

1
Validatequestion exists, choice exists, multi-select rules pass
2
Build internal patternkept for admin/logs, hidden from public client
3
Sum choice scoreseach selected choice adds to user vector
4
Clamp vectorall values stay inside -10 to +10
5
Match perfume DNAlowest weighted distance becomes winner
Distance model

Core vs Meta

Core axes มีน้ำหนักเต็ม ส่วน meta axes ใช้ weight ต่ำกว่าเพื่อให้เป็นบุคลิกเสริม ไม่ override กลิ่นหลักมากเกินไป

Core axes
1.0
Meta axes
0.5
Clamp
-10/+10
Current engine

Weighted vector matching

ระบบปัจจุบันให้คะแนนแต่ละ choice ลง 14 axes แล้วเทียบกับ perfume DNA ของทุกขวด โดย Q3-Q6 เป็น multi-select จึงควรคุม score ต่อ choice ให้อ่อนลง เพื่อไม่ให้คำถามหนึ่งข้อ overpower ผลลัพธ์ทั้งหมด.

  • Core params ใช้เป็นกลิ่น/บุคลิกหลักของ match
  • Meta params เป็น context เสริม เช่น budget/rich/occasion
  • Easter egg ตรวจเงื่อนไขพิเศษก่อนส่งผลปกติ
Legacy from project-spec

Pattern mapping is internal

สเปกเดิมใช้ buildPattern() แล้ว match แบบ Exact → Wildcard * → Default. ในเวอร์ชันปัจจุบัน pattern ยังมีประโยชน์กับ admin/debug/session history แต่ไม่ใช่สิ่งที่ client ควรเห็นบน result page.

  • Example: 1A2B3C4D5A
  • Wildcard mapping เก็บเป็น backward-compatible concept
  • Public payload ต้องซ่อน pattern code หลังคำว่า result/copy ทั้งหมด

11 · Privacy & Tracking

PDPA behavior from project-spec
Collect only when needed

Allowed data

  • Display name / username alias typed by user
  • Email only when user chooses to submit it
  • Quiz answers, internal pattern, scoring vector and result payload
  • Tracking events only after accepted consent
  • Referrer and user-agent for debugging/analytics context
Explicitly out of scope

Never collect

  • Real name, national ID, phone number or sensitive personal categories
  • GPS, precise location or aggressive IP geolocation
  • Device fingerprint, canvas hash or third-party cross-site tracking by default
  • Tracking payload when consent is rejected or withdrawn
1
First visit

Consent banner appears before optional tracking starts.

2
Decision

User accepts or rejects; state is stored client-side.

3
Audit

/api/consent writes consent log with timestamp.

4
Track

If accepted, allowed events go through /api/tracking.

5
Suppress

If rejected, event calls are skipped or ignored.

6
Withdraw

Privacy path can update state to withdrawn when implemented.
