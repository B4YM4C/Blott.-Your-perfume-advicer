01 · Brand CI

Corporate Identity

Brand Positioning

Blot. คือผู้ช่วยเลือกน้ำหอมที่ให้ความรู้สึกเหมือน editorial concierge มากกว่า quiz ทั่วไป: นิ่ง, มีรสนิยม, ไม่ตะโกนขาย, และทำให้ผู้ใช้รู้สึกว่ากลิ่นที่ได้เป็นของเขาจริงๆ.

  • Personality: refined, observant, intimate, slightly playful.
  • Promise: One dip. One match.
  • Primary audience: คนที่อยากหาน้ำหอมให้เหมาะกับบุคลิก งบ และ lifestyle โดยไม่ต้องรู้ note ลึกมาก่อน.
  • Product mood: luxury-lite, accessible, editorial, Bangkok-aware.

Voice & Copy Rules

ภาษาควรสั้น มี texture และไม่เป็น sales copy หนักเกินไป ใช้คำไทยที่อ่านง่ายคู่กับ English cue ที่เป็นแบรนด์.

  • ใช้ sentence สั้นและมีจังหวะ เช่น “Found in a few dips.”
  • หลีกเลี่ยงการเคลมเกินจริง เช่น “ดีที่สุดตลอดกาล” หรือ “เหมาะกับทุกคน”.
  • Result copy ต้องอธิบายเหตุผลจากคำตอบ ไม่ใช่แค่โชว์ชื่อขวด.
  • Admin copy ใช้ภาษาตรงและ functional กว่า public site.

02 · Design Principles

Decision filters
01

Editorial Quiet

ใช้ whitespace, high contrast และ type hierarchy ให้เหมือนนิตยสารน้ำหอม ไม่ใช่ dashboard marketing.

02

One Clear Action

ทุกหน้าควรมี action หลักเดียว: start quiz, choose answer, submit, view result, edit admin data.

03

Tactile Choices

Choice card ต้องแตะง่าย เห็นสถานะชัด รูป outfit ต้องช่วยเลือก ไม่แย่ง attention จากคำถาม.

04

Mobile First Reading

มือถือซ่อน media หนัก, menu เป็น hamburger, section สำคัญขึ้นก่อน, footer สั้นและแบ่งกลุ่ม.

03 · Logo System

Usage, spacing, mistakes

Logo Usage

  • Primary: black logo on paper/offwhite background.
  • Inverse: white logo on ink background.
  • Clear space: อย่างน้อย 1 เท่าความสูงตัว B รอบ logo.
  • Minimum size: 84px width บน mobile header, 120px+ สำหรับ footer/presentation.

Do / Don't

Do

  • วางบนพื้นเรียบ contrast สูง.
  • ใช้สัดส่วนเดิมจาก /public/site-logo.svg.
  • ใช้ inverse เฉพาะบน dark surface.

Don't

  • อย่ายืด บีบ หมุน หรือใส่เงา.
  • อย่าใช้บนภาพที่ busy จนอ่านไม่ออก.
  • อย่า recolor เป็นสี accent ยกเว้น campaign ที่กำหนดใหม่.

Usage Ratio

แบรนด์ควรอ่านเป็น black/white editorial ก่อน แล้วใช้ accent เพื่อแยกข้อมูลหรือทำ presentation ไม่ใช่ตกแต่งทั่วหน้า.

80% paper/offwhite 15% ink/grey 5% accent

Semantic Colors

  • Success: #0a7a3a for admin save/import success only.
  • Error: #b91c1c with pale red surface for destructive/error states.
  • Selected: ink border + paper/ink inversion, not bright color.
  • Disabled: opacity reduction and no hover lift.

Choice must be scannable, touch-friendly and stable in size. Outfit images stay centered and uncropped.

B
Sport

Result Card

Perfume image + fragrance name + notes + reason. Internal pattern code is hidden from public client.

  • Image not decorative
  • Notes visible
  • Reasons explain match

Admin Table

Dense but quiet: offwhite header, mono labels, paper rows, clear row actions.

  • Fast scanning
  • No marketing hero
  • Bulk import/export visible

Alerts

Use semantic color sparingly. Error red only for blocking issue/destructive warning; success green only after save/import.

  • Success: green text or bordered box
  • Error: pale red surface + red border

10 · Implementation Notes

How design lives in the codebase

Code Ownership

  • app/globals.css: design tokens, base components, responsive public layout.
  • app/components/Header.jsx: sticky header, nav, hamburger menu.
  • app/components/Footer.jsx: global footer structure and legal/contact grouping.
  • app/quiz/QuizClient.jsx: quiz choice layout, auto-scroll, Q3 images.
  • app/result/ResultClient.jsx: result card, sound, reasons, alternatives.
  • lib/theme.js: admin editable theme map into CSS variables.

Governance

  • สีหลักต้องมาจาก CSS variables ก่อน hard-code.
  • UI ใหม่ควร reuse .btn, .card, .input pattern.
  • Admin theme editor เปลี่ยน token ได้ แต่ควรจำกัดให้ไม่ทำลาย contrast.
  • ก่อน deploy ให้ตรวจ mobile quiz/result และ admin editor ด้วยตาเสมอ.
